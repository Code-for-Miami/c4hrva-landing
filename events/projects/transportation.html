National Day of Civic Hacking 2015

June 6th | 9:00 a.m. to 6:30 p.m.
The Lab Miami, Wynwood

Join us for Miami's third annual National Day of Civic Hacking event, and help us #hackforchange!

RSVP

Transportation

Each day, South Florida comes together in a myriad of ways, and all through various types of transportation. Join us to brainstorm and collaborate on ways for us to improve the way we visualize, utilize, plan for and communicate about the way we get around Miami-Dade.

Challenges:

  • How do we change the perception of public transit? Contrary to popular belief, public transit is widely used in Miami, but is thought of more as a social service. What can be done to change these perceptions?

  • What if all Miami-Dade regional transit was a unified, cohesive system? There are county-run Metrorails, Metromovers and busses, not to mention trolley buses managed and run by municipalities (and not on Google Transit). Because there are few maps show all modes of transportation in one place, it’s not easy to know if you can get from points A to B through public transportation.

    How could we visualize these options so Miamians can make informed decisions on how to best get to work using multiple agencies? How would all the agencies look on one map?

    • A Code for Miami volunteer built this prototype (Github): could we iterate on this and make it more useful for everyday Miamians?

  • A few county bus lines and municipal trolleys do have real-time tracking, but they are not in GTFS-RT format. Convert those APIs into a GTFS-RT format so we can spin up an experimental deployment of OneBusAway, an open-source bus prediction system.

    Note: There is a pull request open for the proxy transit API to allow for GTFS-RT, but it looks like some conversations still need to happen as far as formatting.

    “Matt Hampel put together a basic map that hits the Detroit Department of Transit (DDOT) OneBusAway API (Github). It's delay-focused, partly to help us see if the system is working properly, partly to help DDOT folks see an overview of the system, but it might be a handy set of calls to the API to use as an example.”

Available data:

  • A wiki of municipal shuttles, and the non-realtime GTFS of Miami-Dade Transit
  • Brigade created API which provides transit data for ten real-time busses in Miami-Dade County and trolleys for the City of Miami. (Github)
  • Transit related information from the Miami-Dade GIS portal
